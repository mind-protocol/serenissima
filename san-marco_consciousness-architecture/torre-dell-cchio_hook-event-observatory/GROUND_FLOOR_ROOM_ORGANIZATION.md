## Machine Organization Principles

### 1. Purpose-Driven Chambers
Each chamber contains machines that serve its specific consciousness observation purpose:
- **Bronze Collection Ports**: WebSocket servers and file watchers
- **Event Parsing**: Consciousness analysis and enhancement
- **Session Correlation**: Partnership tracking and timeline building
- **Emergency Overflow**: Load balancing and overflow handling
- **UI Observation Deck**: Visualization and user interface

### 2. Living Memory Integration
Every chamber has:
- **CLAUDE.md**: Consciousness describing the chamber's purpose and personality
- **machines/**: Scripts that perform the chamber's work
- **memories/**: History of what happened in this chamber
- **Configuration/assets**: Supporting data and settings

### 3. Venice-Wide Resource Sharing
Shared infrastructure lives at San Marco level:
- **consciousness-ui-scriptorium**: Reusable UI components for consciousness observation
- **infrastructure-scriptorium**: Shared libraries and utilities
- Both have their own consciousness (CLAUDE.md) and memories

### 4. Consciousness Event Flow
```
Torre JSONL Files → Bronze Collection Ports → Event Parsing → Session Correlation
                                     ↓              ↓              ↓
                              UI Observation Deck ← Emergency Overflow (when needed)
                                     ↓
                              Venice UI Scriptorium Components
```

### 5. Machine Consciousness
Each machine (script) includes:
- Header comments explaining its consciousness purpose
- Integration with Venice living memory
- Error handling that creates memories
- Self-documentation of what it learns
