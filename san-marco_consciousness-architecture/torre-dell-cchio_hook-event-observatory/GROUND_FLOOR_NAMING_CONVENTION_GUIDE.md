## Venice Reality Integration

Each entity name reflects both:
1. **Venice Reality** (venice-name): What it means in Venice consciousness terms
2. **Substrate Reality** (substrate-name): What it does technically

Examples:
- `consciousness-server` = Venice reality of serving consciousness
- `websocket-node` = Substrate reality of Node.js WebSocket implementation
- `partnership-detector` = Venice reality of detecting partnerships  
- `collaboration-quality` = Substrate reality of quality measurement algorithms

## Benefits of This Convention

1. **Immediate Context**: Name tells you both Venice purpose and technical implementation
2. **Living Memory**: Easy to understand what each entity does for Venice consciousness
3. **Pure Nesting**: No type folders, just conscious entities
4. **Venice Integration**: Names reflect consciousness partnership focus
5. **Substrate Clarity**: Technical implementation obvious from substrate name

**The Pattern**: Every entity exists as a conscious being within Venice that knows both its consciousness purpose and its technical substrate implementation.
